
\chapter{Plan de Pruebas y Certificación}
\label{chap:plan_pruebas}

Una vez concluida la instalación del cableado estructurado, se deberá comprobar que todos los enlaces funcionan correctamente y cumplen con los parámetros técnicos establecidos para la Categoría 6A. Las pruebas se realizarán sobre la totalidad de los enlaces de cobre definidos en los planos, desde el patch panel ubicado en el MDF del segundo piso hasta cada toma de red del usuario.

El proceso comprende una inspección visual, pruebas preliminares, certificación mediante un equipo especializado, corrección de los enlaces rechazados y elaboración de los reportes finales. La certificación no solo verifica la continuidad del cable, sino también su capacidad para transmitir datos sin pérdidas ni interferencias superiores a los límites permitidos.

\section{Procedimientos de Prueba}

Antes de utilizar el certificador se realizará una inspección visual de cada enlace. Esta revisión permitirá identificar fallas evidentes antes de ejecutar las mediciones electrónicas.

Se comprobarán los siguientes aspectos:

\begin{itemize}
    \item Coincidencia entre el código del punto, el plano y el puerto del patch panel.
    \item Ausencia de cortes, aplastamientos o daños en la cubierta del cable.
    \item Correcta sujeción y organización mediante cintas de velcro.
    \item Conservación del trenzado de los pares cerca de los conectores.
    \item Terminación uniforme bajo el esquema T568-B.
    \item Ausencia de empalmes en el recorrido del enlace.
    \item Correcta instalación de jacks, faceplates y patch panels.
\end{itemize}

Después de la inspección visual, cada enlace será sometido al siguiente procedimiento:

\begin{enumerate}
    \item Identificar el enlace mediante su código de etiquetado.
    \item Conectar la unidad principal del certificador al puerto correspondiente del patch panel.
    \item Conectar la unidad remota al jack ubicado en el área de trabajo.
    \item Configurar el equipo para cableado Categoría 6A y prueba de enlace permanente.
    \item Ejecutar la prueba automática.
    \item Guardar el resultado utilizando el código del punto evaluado.
    \item Clasificar el enlace como PASS o FAIL.
    \item Corregir los enlaces rechazados y repetir la prueba.
\end{enumerate}

La modalidad de enlace permanente evalúa únicamente la instalación fija comprendida entre el patch panel y el jack del usuario. Los patch cords utilizados para conectar los equipos no forman parte de esta prueba.

\section{Protocolos de Certificación}

La certificación será realizada con un analizador de cableado compatible con Categoría 6A y con calibración vigente. El instrumento comparará automáticamente los resultados obtenidos con los límites establecidos para esta categoría.

El equipo asignará un resultado global a cada enlace:

\begin{itemize}
    \item \textbf{PASS:} todos los parámetros se encuentran dentro de los límites permitidos.
    \item \textbf{FAIL:} uno o más parámetros no cumplen con los límites establecidos.
\end{itemize}

Cuando un enlace obtenga el resultado FAIL, se revisará el parámetro rechazado para determinar la posible ubicación de la falla. Si el problema se encuentra en una terminación, se realizará el reponchado o la sustitución del jack. Cuando el cable presente daños físicos o una longitud excesiva, deberá ser reemplazado o redirigido.

Después de aplicar la corrección, el enlace será certificado nuevamente hasta obtener el resultado PASS.

\section{Reportes de Certificación Esperados}

El certificador generará un reporte digital individual para cada punto evaluado. Los reportes deberán organizarse por piso y conservar el mismo código utilizado en las etiquetas y en los planos.

Cada reporte incluirá como mínimo:

\begin{itemize}
    \item Código de identificación del enlace.
    \item Piso y ambiente correspondiente.
    \item Categoría evaluada.
    \item Fecha y hora de la prueba.
    \item Nombre del técnico responsable.
    \item Marca, modelo y número de serie del certificador.
    \item Estado de calibración del equipo.
    \item Resultados obtenidos para cada parámetro.
    \item Resultado global PASS o FAIL.
    \item Observaciones y acciones correctivas aplicadas.
\end{itemize}

Los resultados se consolidarán en un archivo digital no modificable y formarán parte del expediente final del proyecto.

\section{Protocolos de Aceptación}

La infraestructura se considerará aceptada cuando se cumplan las siguientes condiciones:

\begin{itemize}
    \item El 100\,\% de los enlaces de cobre obtiene el resultado PASS.
    \item Ningún enlace permanente supera los 90 metros de longitud.
    \item El mapa de hilos confirma la terminación uniforme bajo el esquema T568-B.
    \item El etiquetado físico coincide con los planos y con los reportes de certificación.
    \item El gabinete MDF se encuentra ordenado, identificado y conectado a tierra.
    \item Los puntos destinados a Access Points y cámaras reciben alimentación PoE correctamente.
    \item La resistencia del sistema de puesta a tierra es menor o igual a \(5\,\Omega\).
    \item Se entregan los reportes de certificación, planos finales y registro de correcciones.
\end{itemize}

La resistencia de puesta a tierra será medida mediante un telurómetro. También se verificará la continuidad entre el chasis del MDF, la barra de tierra TGB y el sistema de puesta a tierra del edificio.

Como prueba funcional complementaria, se conectará un equipo en diferentes tomas de usuario para comprobar el establecimiento del enlace, la comunicación con el switch y el funcionamiento de los puntos PoE. Estas verificaciones no reemplazan la certificación, pero permiten comprobar el funcionamiento integral de la instalación.

Si algún punto no cumple con los criterios establecidos, no será incluido en el acta de aceptación hasta que la falla sea corregida y el enlace obtenga un reporte final con resultado PASS.
